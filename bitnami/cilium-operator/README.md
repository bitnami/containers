Non-root container images add an extra layer of security and are generally recommended for production environments. ## Get this image

The recommended way to get the Bitnami Cilium Operator Docker Image is to pull the prebuilt image from the [Docker Hub Registry](https://hub.docker.com/r/bitnami/cilium-operator).

```console
docker pull bitnami/cilium-operator:latest
```

To use a specific version, you can pull a versioned tag. You can view the [list of available versions](https://hub.docker.com/r/bitnami/cilium-operator/tags/) in the Docker Hub Registry.

```console
docker pull bitnami/cilium-operator:[TAG]
```

If you wish, you can also build the image yourself by cloning the repository, changing to the directory containing the Dockerfile and executing the `docker build` command. Remember to replace the `APP`, `VERSION` and `OPERATING-SYSTEM` path placeholders in the example command below with the correct values.

```console
git clone https://github.com/bitnami/containers.git
cd bitnami/APP/VERSION/OPERATING-SYSTEM
docker build -t bitnami/APP:latest .
```

## Configuration

### Running commands

To run commands inside this container you can use `docker run`, for example to execute `cilium-operator-generic help` you can follow the example below:

```console
docker run --rm --name cilium-operator bitnami/cilium-operator:latest help
```

Check the [official Cilium Operator documentation](https://docs.cilium.io/en/stable/internals/cilium_operator/) for more information about configuration options.

### FIPS configuration in Bitnami Secure Images

The Bitnami Cilium Operator Docker image from the [Bitnami Secure Images](https://go-vmware.broadcom.com/contact-us) catalog includes extra features and settings to configure the container with FIPS capabilities. You can configure the next environment variables:

- `OPENSSL_FIPS`: whether OpenSSL runs in FIPS mode or not. `yes` (default), `no`.
